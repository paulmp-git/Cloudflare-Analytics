# Contributing to Cloudflare Analytics

Thank you for your interest in contributing to Cloudflare Analytics! We welcome contributions from the community.

## Bug Reports

If you find a bug, please create an issue in the repository with the following information:
- A descriptive title
- Detailed steps to reproduce the issue
- Your environment details (WordPress version, PHP version, Plugin version)
- Error logs (if applicable)

## Feature Requests

We are open to new ideas! If you have a feature request, please open an issue and describe:
- The feature you would like to see
- The problem it solves
- Any specific implementation details you have in mind

## Pull Requests

1.  Fork the repository.
2.  Create a new branch for your feature or bug fix (`git checkout -b feature/amazing-feature`).
3.  Make your changes.
4.  Commit your changes with descriptive commit messages.
5.  Push to your branch (`git push origin feature/amazing-feature`).
6.  Open a Pull Request.

## Coding Standards

- Please follow the [WordPress Coding Standards](https://developer.wordpress.org/coding-standards/).
- Ensure your code is secure and follows best practices.
- Test your changes thoroughly before submitting.
